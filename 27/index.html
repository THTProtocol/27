Match Lifecycle
Create Match

A player creates a match by selecting a game, setting the escrow amount, and choosing time controls. The escrow is locked into a UTXO covenant on the Kaspa blockchain.

Details
Accept Match

An opponent accepts the match by matching the escrow amount. Both escrows are now locked, and the game begins once both deposits are confirmed on-chain.

Details
Play Game

Players compete in real-time with moves synchronized through Firebase. Game state is validated continuously and time controls are enforced server-side.

Details
Resolve Outcome

The oracle daemon verifies the game outcome, signs the result, and submits it to the covenant. A dispute window opens for challenges before finalization.

Details
Settle Escrow

The escrow covenant releases funds to the winner. Settlement is automatic and trustless, requiring only valid oracle signatures to execute the payout transaction.
