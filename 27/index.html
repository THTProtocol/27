Match Lifecycle

Create

Match created - escrow address generated via covenant

Details
Creator selects game type, stake amount, and time control. A unique P2SH escrow address is derived from the match parameters. The creator sends their stake to this address as the first funding transaction.

Accept

Opponent joins and locks matching escrow

Details
The opponent sends a matching stake to the same escrow address. Both UTXOs are now locked. The match becomes active once both transactions reach sufficient DAA confirmations.

Play

Game in progress with synchronized clocks

Details
Moves are relayed via Firebase Realtime Database for low-latency sync. Each player has a local clock enforced by the UI. Move validation runs client-side (chess.js, custom engines for Connect4/Checkers). The board state is signed per-move for oracle verification.

Resolve

Oracle validates result and signs attestation

Details
The bonded oracle network verifies the game result by replaying signed moves. A SHA-256 commit-reveal scheme prevents front-running. The oracle signs an attestation message that unlocks the covenant spending path for the winner.

Settle

Winner receives payout from escrow (2% protocol fee)

Details
The auto-payout engine constructs the settlement transaction using the oracle attestation signature. 98% goes to the winner, 2% is the protocol fee. Settlement is typically confirmed within one DAA period (about one second on Kaspa).
