How It Works

Create
Lock KAS in an on-chain escrow
Challenger sends KAS to a P2SH escrow address derived from both player public keys. Transaction is broadcast to Kaspa TN12 and confirmed within 1 second.
Accept
Opponent matches the escrow
Joiner matches the escrow amount into the same address. The covenant now holds 2x the stake. Firebase records both UTXO references.
Play
Every move hashed and committed on-chain
Every move is hashed client-side using SHA-256 and the root is committed via OP_RETURN in a zero-value transaction. This creates an immutable audit trail of game state.
Resolve
Oracle verifies the winner
The oracle daemon monitors the match document. On game completion, it verifies the winner signature and confirms the escrow UTXOs are unspent.
Settle
Winner receives KAS directly
A payout transaction sweeps both escrow UTXOs, sends 98% to the winner address and 2% to the protocol treasury. No withdrawal needed: funds arrive in the winner's wallet.
